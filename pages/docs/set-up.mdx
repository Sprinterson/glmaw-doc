# Installation
Comment installer l'Ads Wrapper ?

## Script fourni
Le Wrapper se présente sous la forme d'une URL, se terminant par `glmaw.js`.

## Pré-requis
Il convient qu'une variable globale nommée `"ADS_CONFIG"`, sous forme d'objet, soit exposée dans le code HTML de la page, afin que le Wrapper puisse récupérer les informations concernant l'**Ad Unit** de la page (indispensable) et éventuellement d'autres options comme les **targets** (optionnel).

```html
<script>
let ADS_CONFIG = {
  "adUnit:" "/gam/global/path/",
  "targeting:" {"keyword:" "a_la_une"},
}
</script>
```

## Script du Wrapper
Le script du Wrapper, `glmaw.js`, doit être chargé soit après consentement de l'utilisateur via la CMP, soit s'il a déjà consenti par le passé. Le Wrapper traite alors les informations transmises par `"ADS_CONFIG"` afin de correctement émettre les requêtes et l'affichage des pubs.

## Positions publicitaires
Une fois le script et la configuration en place, il suffit d’ajouter des marqueurs HTML aux emplacements où doivent se trouver les publicités.

```html
<div id="pave_haut" class="dfp-slot" data-format="pave_haut"></div>
```

- Une **classe HTML** commune à toutes les positions est **obligatoire** (le nom de cette classe est à la préférence des développeurs de l'application, mais doit être précisée aux développeurs du Wrapper pour la pré-configuration de celui-ci), c’est ce qui permet au Wrapper de retrouver les emplacements publicitaires présents dans la page.
- L’identifiant est facultatif, le Wrapper en calcule un automatiquement s’il n'est pas renseigné mais il préférable d'en préciser un.
